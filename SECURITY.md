# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in strux, please report it privately
so it can be fixed before public disclosure. **Do not file a public GitHub
issue for security reports.**

- **Email:** info@gesp.tech
- **Subject prefix:** `[strux security]`

Please include:

- Affected module(s) (`core`, `adapter-minecraft`) and commit/version.
- Steps to reproduce.
- Impact assessment — what an attacker could do (server crash, world
  corruption, RCE, etc.).
- Any suggested mitigation or patch.

I aim to reply within a week or two, and to investigate and patch confirmed
vulnerabilities promptly. Once a fix is released, reporters are credited in the
changelog unless they prefer to remain anonymous.

## Supported Versions

strux is released at 1.0.0. Security fixes target the latest 1.x release; please
upgrade to the newest 1.x version before reporting.
